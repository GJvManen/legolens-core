# Contributing

Thanks for contributing to LegoLens.

## Ground rules

- Keep the project review-first.
- Do not add secrets, API keys or private data.
- Add tests for workflow or data-model changes.
- Update docs when adding features.
- Use case packs for case-specific data.

## Suggested workflow

1. Open an issue.
2. Create a feature branch.
3. Add tests and docs.
4. Run `npm run build`, `npm run validate`, `npm test`.
5. Open a pull request.

## Commit style

Use clear commits, for example:

```text
feat(case-pack): add demo case manifest
fix(review): preserve approval audit trail
docs(github): add push guide
```
